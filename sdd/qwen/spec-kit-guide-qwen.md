# 초보자를 위한 spec-kit을 이용한 SDD 개발 방법 가이드

## 1. SDD(Specification Driven Development)란?

SDD(Specification Driven Development, 사양 주도 개발)는 전통적인 코딩 방식과는 반대되는 접근법입니다. 일반적으로는 먼저 코드를 작성하고 그에 따라 요구사항을 정의하지만, SDD는 **먼저 명확한 사양(specification)을 작성한 후**, 그 사양에 따라 구현을 진행합니다.

SDD의 핵심 철학은 다음과 같습니다:
- 사양이 "무엇을"(what) 만들어야 하는지 정의
- 기술 계획이 "어떻게"(how) 구현할지 결정
- 구현은 사양에 따라 자동화될 수 있음

GitHub의 spec-kit은 이러한 SDD 방법론을 지원하는 오픈소스 툴킷으로, AI 코딩 에이전트와 함께 사용하여 명확한 지침을 기반으로 코드를 생성할 수 있게 해줍니다.

## 2. spec-kit 설치 및 준비

SDD 개발을 시작하려면 먼저 spec-kit을 설치해야 합니다. 다음 조건이 필요합니다:
- Python 3.11 이상
- uv 패키지 매니저
- Git

```bash
# uv 설치 (패키지 매니저)
pip install uv

# spec-kit CLI 설치
uv tool install specify
```

## 3. SDD 개발 단계별 설명

SDD 개발은 5단계로 구성됩니다. 각 단계를 차례대로 진행하면서 사양 중심 개발을 구현할 수 있습니다.

### 단계 1: 프로젝트 헌법(Constitution) 작성

**목표**: 프로젝트의 기본 원칙과 규칙을 정의

헌법(constitution) 문서는 프로젝트의 비협상적 원칙과 조직의 규칙을 정의합니다. 이 문서는 모든 후속 사양과 구현이 따라야 할 기본 규칙을 담고 있습니다.

```bash
# 새 프로젝트 초기화
specify init my-project
cd my-project

# 헌법 생성
specify constitution
```

헌법에는 다음과 같은 내용이 포함됩니다:
- 코딩 스타일 및 형식 지정 규칙
- 보안 요구 사항
- 기술 스택 제한
- 라이센스 및 컴플라이언스 요구사항
- 테스트 커버리지 기준
- 코드 리뷰 프로세스

### 단계 2: 사양(Specification) 작성

**목표**: 무엇을 만들지 명확히 정의

사양(spec) 문서는 기능 요구사항과 사용자 여정을 설명합니다. 이 단계에서는 "어떻게" 구현할지는 고려하지 않고, "무엇을" 만들어야 하는지만 정의합니다.

```bash
# 사양 생성
specify specify "사용자가 로그인할 수 있는 기능"
```

사양 작성 시 고려할 사항:
- 사용자 시나리오 설명
- 기능의 목적과 배경
- 예외 상황 및 제외 사항
- 성공 및 실패 기준
- 입력 및 출력 정의

### 단계 3: 계획(Plan) 수립

**목표**: 사양을 기반으로 기술적 구현 계획 수립

계획(plan) 문서는 사양을 기반으로 기술 스택, 아키텍처, 데이터 계약 등을 결정합니다. 헌법 문서의 규칙을 준수해야 합니다.

```bash
# 계획 생성
specify plan
```

계획에는 다음과 같은 내용이 포함됩니다:
- 사용할 기술 스택
- 아키텍처 패턴
- 데이터베이스 설계
- API 인터페이스
- 보안 구현 방법
- 테스트 전략

### 단계 4: 작업(Task) 분할

**목표**: 구현을 위한 구체적인 작업 항목으로 분할

작업(tasks) 문서는 계획을 기반으로 실제 구현을 위한 작은 작업 단위로 나눕니다.

```bash
# 작업 분할 생성
specify tasks
```

작업 분할 시 고려할 사항:
- 구현 순서
- 의존성 관계
- 테스트 우선 구현 여부
- 코드 리뷰 및 검토 단계
- 배포 계획

### 단계 5: 구현(Implementation)

**목표**: 사양, 계획, 작업을 기반으로 실제 코드 구현

```bash
# 구현 실행
specify implement
```

AI 코딩 에이전트는 사양, 계획, 작업 문서를 기반으로 코드를 생성합니다. 이 과정에서 인간 개발자는 검증과 품질 보증 역할을 수행합니다.

## 4. SDD 개발의 장점

### 명확한 요구사항 정의
- 사양 중심 개발로 "vibe coding"(감에 의한 코딩) 방지
- 명확한 사양이 있어 AI 에이전트에게 정확한 지침 제공
- 요구사항 불일치 문제를 구현 전에 발견

### AI 활용 극대화
- AI는 명확한 지침이 있을 때 가장 잘 작동
- 사양을 기반으로 코드 생성 시 일관성 향상
- 다양한 구현 옵션 탐색 가능

### 유지보수 용이성
- 사양 문서는 살아있는 문서로 프로젝트와 함께 진화
- 코드 변경 시 사양과 비교하여 일관성 유지
- 새로운 팀원에게 빠른 적응 가능

### 팀 협업 향상
- 모든 팀원이 동일한 사양을 바탕으로 작업
- 사양을 기준으로 피드백과 논의 가능
- 역할 분담 시 명확한 기준 제공

## 5. SDD 개발 실습 예제

간단한 웹 애플리케이션을 만드는 예를 들어보겠습니다:

### 예제: 간단한 할일 목록 애플리케이션

#### 1. 헌법 문서 생성
```bash
specify constitution
```
- 백엔드: Python Flask
- 프론트엔드: HTML/CSS/JavaScript
- 데이터베이스: SQLite
- 보안: 입력 검증 필수

#### 2. 사양 문서 생성
```bash
specify specify "사용자가 할일을 추가, 조회, 수정, 삭제할 수 있는 애플리케이션"
```
- 사용자는 할일을 추가할 수 있음
- 사용자는 할일 목록을 조회할 수 있음
- 사용자는 할일을 수정할 수 있음
- 사용자는 할일을 삭제할 수 있음
- 할일은 제목, 내용, 완료 여부 속성을 가짐

#### 3. 계획 문서 생성
```bash
specify plan
```
- Flask를 사용한 REST API 서버
- 간단한 HTML UI
- SQLite 데이터베이스 사용
- GET/POST/PUT/DELETE API 엔드포인트

#### 4. 작업 분할
```bash
specify tasks
```
- 1. 데이터베이스 스키마 설계
- 2. Flask 애플리케이션 구조 설정
- 3. 할일 CRUD API 구현
- 4. 간단한 UI 개발
- 5. 테스트 작성

#### 5. 구현
```bash
specify implement
```
- AI가 사양, 계획, 작업을 기반으로 실제 코드 생성

## 6. SDD 개발 팁

### 사양 작성 팁
- 구체적이고 측정 가능한 언어 사용
- 예외 상황과 제외 사항 명확히 정의
- 사용자 시나리오 중심으로 작성
- 비즈니스 목적과 기술 목적 분리

### 계획 수립 팁
- 헌법 문서의 제약 조건 준수
- 확장성과 유지보수성 고려
- 보안 요구사항 반영
- 테스트 가능성 고려

### 협업 팁
- 사양 문서에 대한 정기 리뷰
- 변경 사항 발생 시 사양 먼저 업데이트
- 팀원 모두가 사양 문서에 접근 가능하도록 관리
- 사양과 구현 간 차이점 모니터링

## 7. 결론

spec-kit을 이용한 SDD(Specification Driven Development)는 AI 시대에 매우 효과적인 개발 방법론입니다. 명확한 사양을 기반으로 하여 AI 코딩 에이전트와 효율적으로 협업할 수 있으며, 코드의 일관성과 품질을 높일 수 있습니다.

초보자라도 위의 5단계를 순서대로 따라가면 SDD 개발을 시작할 수 있습니다. 중요한 것은 사양을 충분히 정의한 후에 구현에 들어가는 것입니다. 이를 통해 개발 과정의 불확실성을 줄이고, 보다 예측 가능한 결과를 얻을 수 있습니다.
