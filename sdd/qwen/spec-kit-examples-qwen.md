# GitHub spec-kit 사용 예제 저장소 추천

GitHub spec-kit 템플릿(structure: spec.md, plan.md, constitution.md, tasks.md, checklist.md)을 사용하는 데 도움이 될 수 있는 인기 있는 저장소를 별점이 많은 순서대로 추천합니다. 실제 spec-kit 템플릿을 사용하는 저장소는 많지 않지만, 사양 주도 개발(Specification Driven Development) 방법론을 따르는 인기 있는 저장소를 소개합니다.

## GitHub spec-kit 템플릿 사용 팁

GitHub spec-kit의 템플릿을 사용할 때 다음 순서로 문서를 작성하는 것이 좋습니다:

1. **spec.md** - 프로젝트의 기본 사양과 요구사항을 정의
2. **plan.md** - 사양을 기반으로 개발 계획 수립
3. **tasks.md** - 구체적인 작업 항목으로 분할
4. **checklist.md** - 완료 기준 및 검증 절차 정의
5. **constitution.md** - 프로젝트 거버넌스 및 의사결정 구조 정의

이러한 순서로 문서를 작성하면 명확한 구조를 가진 사양 주도 개발을 구현할 수 있습니다. 위의 추천 저장소들은 실제 사양 주도 개발이 어떻게 구현되고 활용되는지를 보여주는 좋은 참고 자료가 될 것입니다.
